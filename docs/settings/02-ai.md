# AI 模块

## 模块目标

`AI` 是设置页的一等模块，负责管理模型接入、默认运行配置、模型候选集，以及 AI 配置的导入导出。

这部分要尽量贴近当前平台已存在的 `AIConfigEntry`、全局 AI store 和设置页交互，而不是重新定义一套脱离代码的数据结构。

## 范围边界

`AI` 负责：

- Provider 配置
- API Key 与 Base URL
- 默认模型与候选模型
- 配置启停与默认选择
- 连接测试
- 配置导入导出

`AI` 不负责：

- 工具审批与沙箱，归 `权限`
- MCP Server 连接，归 `MCP 服务器`
- Shell 模式与 Claude / Codex 绑定，归 `高级`

## 子分组

1. 配置列表
2. Provider 与凭据
3. 模型
4. 运行默认值
5. 导入导出

## 关键行为补充

- `AIConfigEntry` 是当前最重要的代码事实来源，后续设置模型不要偏离它。
- `provider` 切换后，`baseURL`、endpoint 预览、候选模型策略要联动刷新。
- `savedModels` 不能只是展示缓存，它还承担活动模型回退保护。
- `enabled` 和 `selectedConfigId` 不是同一件事：前者决定可用性，后者决定默认运行配置。
- 导入时建议先写入为禁用态，避免一导入就污染当前运行配置。

## 功能清单

1. 配置列表与 CRUD：查看、新建、编辑、删除、搜索多条 AI 配置。
2. Provider 与凭据：编辑 `provider`、`apiKey`、`baseURL`、`customHeaders`，并给出 endpoint 预览。
3. 模型管理：编辑 `model`、维护 `savedModels`、拉取模型列表、设置 `contextWindowTokens`。
4. 连接验证：支持连接测试，并以“测试中 / 成功 / 失败”三态反馈。
5. 默认配置：支持切换 `selectedConfigId`，并在后续扩展场景级模型覆盖。
6. 导入导出：支持复制 JSON 导出与 JSON 导入，并保留格式版本字段。
