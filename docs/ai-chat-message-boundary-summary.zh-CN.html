消息边界问题，一页看懂

现在最核心的问题，不是某个样式没调好，而是 “过程反馈”和“最终正文”没有被严格分开。 所以才会出现过程和结果不一致、上一条消息被下一条影响、反馈重复出现这些问题。

当前最大问题：正文定义混了 思考更像状态，不该入正文 工具链路较清楚，但还有旧兼容残留 最优路线：只保留一个正文真相源
一句话结论

现在不是“显示有点怪”，而是“正文到底是什么”这件事还没有收口。

最适合 agent 的方向

过程只显示状态和工具，最终只保留一份最终正文，不再多源拼装。

这页的目标

只讲你真正关心的产品语义，不先讲英文类型名和接口细节。

先说人话：现在到底乱在哪里

1

过程和最终共用一类正文

“我先看看”“我正在检查”“这是最后结论”现在本质上都被当成同一种正文，所以系统没法天然分清楚谁该留下、谁只该临时展示。

2

正文同时有两份来源

运行中的正文走一套，完成后的正文又走一套。只要两边不同步，就容易出现过程和结果不一致。

3

实时正文没有强绑定到消息

实时流式内容还在靠“猜它属于哪条消息”，不是运行时直接标记归属，所以会有“影响上文”的风险。

4

旧路径还在兜底

工具和过程展示虽然已经收了一部分，但旧的兼容路径还在，UI 还要靠“压重复”来救场。

三个边界，应该怎么理解

思考

现在更像一种状态。

  • 运行中可以显示“思考中”。
  • 完成后不该再当正文展示。
  • 它不应该参与最终答案的组成。

反馈

现在最乱，问题主要出在这里。

  • 过程话术和最终答案都混在正文里。
  • 系统缺少单独的“过程反馈”定义。
  • 这直接导致重复、串位、不一致。

工具

方向基本对，但还没彻底收口。

  • 工具应只认运行时事件。
  • 旧文本解析只保留兼容用途。
  • 不该再作为实时展示主来源。

为什么你会看到这些具体问题

反馈出现两遍

因为过程话术和最终正文都被当成正文，而且正文还有多份来源，只能靠界面层临时压掉一部分重复。

下一问影响上一问

因为实时流式正文没有被运行时直接绑定到某一条消息，界面还在依赖投影结果去反推归属。

过程和结果不一致

因为“流式过程正文”和“最终完成正文”不是同一个真相源，刷新、回放、完成切换时就会出现偏差。

过程里吐出一半一直留着

因为过程文字已经进入正文语义，而不是纯临时展示，所以完成后系统很难自然地把它们清干净。

我建议的唯一主线

不要再东一块西一块补 UI 了。 最合适的路线是： 把“最终正文”重新定义成唯一真相源，再把过程展示全部退回到状态层和工具层。

不推荐继续这样改

  • 发现哪里重复，就在 UI 上再压一次。
  • 发现哪里顺序不对，就再补一个排序规则。
  • 发现哪里串位，就再补一层展示判断。
  • 结果会是：补丁越来越多，问题反复回来。

推荐的最终路线

  • 思考只保留“思考中”状态，不入最终正文。
  • 过程反馈不进最终正文，只做临时展示。
  • 工具只认运行时事件，不再让旧文本路径主导。
  • 最终正文只保留一份，完成前后都从同一来源来。

最优改法，按这四步走

1

先定义“哪些字不是最终正文”

先把过程话术从最终正文里拆出去。也就是把“我先看看”“我正在检查”这类内容降级为临时展示，不再天然落到最终答案里。

2

给实时流式正文加消息归属

让运行时直接说清楚：这段实时正文属于哪条消息。这样才能真正解决“下一条影响上一条”。

3

收掉正文双来源

过程展示和完成展示都从同一份正文语义来。时间、状态、耗时可以单独算，但正文只能有一份。

4

把旧工具文本路径降级成兼容层

老的文本解析不再参与实时主链路，只给历史回放、导入兼容使用，避免继续搅乱现在的时间线。

删什么、改什么、保留什么

建议删除或降级

  • 把过程话术长期保留进最终正文的做法。
  • 让旧工具文本解析参与实时展示主链路。
  • 靠 UI 临时 suppression 维持正确性的逻辑。

必须重点修改

  • 正文边界定义。
  • 实时正文归属关系。
  • 完成前后正文的单源化。
  • 过程展示和最终结果的职责切分。

建议保留

  • 思考只显示“思考中”的产品方向。
  • 工具按事件驱动展示的总体思路。
  • 完成时再显示时间和总结的产品方式。

为什么这条路线最适合 agent

对用户更稳定

用户只会看到两类东西：过程状态 / 工具动作，以及最终正文。体验清晰，不容易前后变来变去。

对系统更稳

把真相源收成一个之后，刷新、恢复、回放、下一轮开始，都不再需要靠界面层猜和补。

对后续迭代更省

以后再做段落流式、块状流式、收口折叠，都是在一个稳定语义上改，而不是到处救火。

对测试更好写

可以直接验证“同一条消息完成后不再变化”“过程与结果来自同一正文语义”，而不是测很多展示细枝末节。
