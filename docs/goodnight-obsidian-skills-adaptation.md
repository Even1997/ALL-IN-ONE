# GoodNight AI Mainline

## Current Direction

GoodNight 的 AI 主线现在只保留三条内建 slash 技能：

- `/wiki`
- `/sketch`
- `/ui-design`

这三条就是默认入口。聊天默认保持自然对话，只有用户显式输入 slash 技能时才进入对应能力。

## User-Facing Rules

- 不再保留任何 `@...` 形式的旧入口。
- 不再把旧的 requirements、knowledge organize、change sync 当成默认聊天技能。
- 不再把隐藏流程、额外壳子或中间产物暴露给用户。
- 不再要求用户理解一套额外的内部链路才能正常使用 AI。

## `/wiki`

`/wiki` 用来处理项目上下文、笔记、页面说明和相关源码的联合阅读。

目标是：

- 优先读取当前项目里已经存在的内容
- 区分已确认事实和待确认信息
- 直接给出 grounded 的回答、整理结果或下一步建议

它不是额外流程入口，也不是另一套独立产品线。

## `/sketch`

`/sketch` 用来输出低保真页面结构、模块拆分和线框级说明。

重点是：

- 结构先于视觉
- 明确页面目标、模块职责和关键信息
- 保持低保真，不直接跳成视觉稿

## `/ui-design`

`/ui-design` 用来在已有结构基础上细化界面层。

重点是：

- 保留已经确认的页面目标和信息层级
- 细化视觉层次、状态、交互和实现提示
- 不在没有说明的情况下重写信息架构

## Internal Principle

内部可以继续做上下文选择、文件检索、技能装配、MCP 调用、读写审批和记忆写回，但这些都应该是实现细节。

用户看到的体验应该始终是：

1. 直接聊天可用
2. 输入 `/wiki`、`/sketch`、`/ui-design` 时能力稳定
3. 读写、技能、MCP、记忆、审批链路尽量低惊扰
4. 没有历史遗留入口把用户带回旧路线

## Cleanup Standard

如果某个入口、文案、状态字段、页面按钮或测试仍然依赖旧技能名、旧入口形式或旧 workflow 壳子，就应该继续删除或改到当前主线。
