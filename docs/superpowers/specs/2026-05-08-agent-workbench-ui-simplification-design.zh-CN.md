# Agent 工作台 UI 精简设计

## 摘要

本设计针对当前 `Agent` 工作台“信息重复、图标过大、入口过多、页面像功能展板而不是聊天工作台”的问题，做一次明确的减法重构。

目标不是更换整套布局语言，也不是新增功能，而是把当前页面收成一个以聊天为主、以最近对话为核心、以轻量审查为辅助的工作台。

本次设计直接删除一批当前价值较低或重复的工作台入口，并把一部分能力改为弹窗或外层系统入口，避免左侧和主舞台继续堆积无效内容。

## 目标

- 把 `Agent` 页面收成一个真正的聊天工作台，而不是多入口功能首页。
- 删除重复内容、重复文案和重复入口。
- 缩小左侧图标与按钮体积，降低“每个入口都像功能块”的压迫感。
- 让 `最近对话` 成为左侧主体内容，而不是被大卡片和说明区挤压到下方。
- 让 `新对话` 点击后直接创建新线程，而不是切换到一个“新对话页面”。
- 让 `搜索`、`技能`、`设置` 回到更符合语义的位置：搜索和技能用弹窗，设置回到外层系统区域。

## 非目标

- 不重新设计整体桌面应用壳层。
- 不改动聊天运行时、线程存储、审查数据模型或消息流。
- 不在这一轮新增新的搜索能力或技能后端能力，只调整入口位置与呈现方式。
- 不把右侧审查区完全移除，仍保留最必要的审查与文件查看能力。

## 当前问题

当前页面的主要问题不是单一布局错误，而是多个轻重不一的 UI 冗余叠加在一起：

- 左侧同时存在品牌块、说明头部、hero 卡、快捷入口卡、最近对话，信息层级重复。
- `新对话` 入口的语义错误，当前更像“切页”而不是“直接创建会话”。
- `插件`、`自动化` 仍占据一级入口，但当前只是占位，不应继续占用注意力。
- `搜索`、`技能`、`设置` 都被塞进工作台结构里，导致左侧既像导航又像控制台。
- 右侧 inspector 同时承载 `审查 / 文件 / 工具 / 记忆 / 上下文`，信息密度过高。
- 顶部状态胶囊较多，和聊天主舞台争夺注意力。
- 左侧入口和页面内部还有重复说明文案与重复功能承接。

## 已确认决策

- 删除左侧 `插件` 入口。
- 删除左侧 `自动化` 入口。
- 左侧保留 `新对话 / 搜索 / 技能`。
- `设置` 从 Agent 工作台移出，交给应用最外层系统区域承接。
- `最近对话` 承接左侧主体内容。
- `搜索` 改为独立弹窗，不再占用左侧工作台主体区域。
- `搜索` 的目标是“系统里的文档内容”，不再是线程过滤页。
- `技能` 改为独立弹窗，不再占用左侧工作台主体区域。
- 右侧 inspector 只保留 `审查 / 文件 / 记忆`。
- 删除 inspector 中的 `工具 / 上下文`。
- `新对话` 点击后直接创建线程并切换进去。

## 设计原则

### 先保留真正高频的事

进入这个页面后，用户最常做的事情应该只有两类：

- 开始一个新对话
- 回到最近的某个对话

任何不能服务这两类核心动作的一级内容，都应该下沉、收纳或删除。

### 同一种能力只保留一个入口

如果页面里已经有 `最近对话` 列表，就不再需要额外的大卡片或快捷卡去重复承接它。

如果 `搜索` 和 `技能` 是独立动作，就不应再占据左侧主体结构，避免导航、内容区和工具页混在一起。

### 把“设置”还给系统层

`设置` 属于环境与应用级能力，不应继续混在 Agent 工作台的日常聊天路径里。Agent 页面保留工作流入口，系统设置回归外层统一承接。

### 聊天舞台优先

中间主舞台必须保持为最强视觉焦点。左侧和右侧都只能为主舞台服务，不能继续发展成独立首页或调试面板。

## 页面结构

### 总体结构

页面保持三栏工作台框架，但显著减法：

- 左侧：轻量入口 + 最近对话
- 中间：聊天主舞台
- 右侧：精简后的审查区

视觉重点从“多卡片并列”改为“线程与聊天并列”。

### 左侧结构

左侧分成两个区块：

- 顶部轻入口区：`新对话 / 搜索 / 技能`
- 主体区：`最近对话`

其中：

- `新对话` 是主动作，位于顶部最优先位置
- `搜索` 和 `技能` 是次级动作，保留在顶部，但不再展开成主体页面
- `最近对话` 占据左侧主要高度，成为真正的线程承接区

### 中间主舞台

中间继续使用现有聊天舞台作为主区域，但移除不必要的外围噪音：

- 删除左侧 hero 卡和快捷入口卡后，中间成为更明确的主角
- 顶部标题区保留必要的会话标题与少量状态
- 不再用过多胶囊状态和装饰性块去抢主舞台注意力

### 右侧 inspector

右侧保留，但只保留真正有价值的三类信息：

- `审查`
- `文件`
- `记忆`

删除：

- `工具`
- `上下文`

这意味着右侧仍然是“辅助面板”，但不再承担完整调试台职责。

## 交互设计

### 新对话

当前左侧 `新对话` 入口从“切到 threads 页面”改为真实动作：

- 点击后立即创建新线程
- 自动切换到这个新线程
- 聊天主舞台直接进入该对话

页面中不再保留额外的“大号新建对话卡片”作为重复入口。

### 最近对话

`最近对话` 直接承接线程列表：

- 列表保留当前线程高亮
- 点击线程进入对应会话
- 有恢复能力的线程仍可继续提供恢复动作

“最近对话”标题保留，但只作为轻量分组标题，不再额外配套说明卡或重复摘要。

### 搜索

`搜索` 从左侧主体页面改为独立弹窗：

- 点击顶部 `搜索` 入口时打开弹窗
- 搜索目标为系统内部文档内容
- 搜索结果在弹窗内展示和跳转

本次只调整入口与容器，不定义新的搜索引擎能力。

### 技能

`技能` 从左侧主体页面改为独立弹窗：

- 点击顶部 `技能` 入口时打开弹窗
- 技能管理仍由现有技能页逻辑承接
- 不再占据左侧主体布局

### 设置

`设置` 完全移出 Agent 工作台：

- Agent 页面不再显示 `设置` 入口
- 相关配置入口交给最外层系统区域承接
- 明暗模式与系统级配置和 Agent 页面解耦

## 视觉精简规则

### 左侧减法

删除以下结构：

- hero 卡
- 快捷入口卡
- 重复说明区
- 占位型 `插件 / 自动化` 面板

保留左侧的唯一叙事：

- 上方开始动作
- 下方最近对话

### 图标与按钮体积

左侧 rail 图标和按钮整体缩小，原则如下：

- 图标尺寸回到“导航图标”尺度，而不是“功能卡图标”尺度
- 按钮高度和内边距减小
- 激活态仍清晰，但不再依赖夸张体积来强调

### 顶部状态区

顶部状态区只保留必要状态，不再出现过多同类状态 pill 并排堆积。

保留原则：

- 当前运行时信息可以保留
- 当前会话状态可以保留
- 明显阻塞态如待审批可以保留

应删除或合并那些只是在重复表达“已连接 / idle / ready”一类含义的状态展示。

## 技术实现边界

本次实现预计只涉及 Agent 工作台相关前端组件与样式，不应扩展到运行时内核：

- `AgentShellPage.tsx`
- `AgentWorkbenchSidebar.tsx`
- `AgentWorkbenchInspector.tsx`
- `agentWorkbench.css`
- 与 `搜索`、`技能` 弹窗承接有关的现有 UI 组件

必要时可调整线程创建触发逻辑，但不应改动线程数据模型。

## 测试策略

至少覆盖以下验证：

- 点击左侧 `新对话` 后会直接创建线程并切换进入该线程
- 左侧不再出现 `插件 / 自动化 / 设置` 入口
- 左侧不再出现 hero 卡、快捷卡、重复说明卡
- `最近对话` 仍能展示线程并进入指定会话
- `搜索` 可以以弹窗方式打开
- `技能` 可以以弹窗方式打开
- 右侧 inspector 不再出现 `工具 / 上下文`
- 右侧仍保留 `审查 / 文件 / 记忆`

如果当前项目已有桌面 UI 测试，应优先用存在的工作台测试方式补充断言，而不是新增一套完全独立的测试框架。

## 范围外

本轮不做：

- 全新的全局搜索产品设计
- 技能页内容重构
- 系统设置页重构
- 右侧 inspector 的进一步合并或彻底移除
- 聊天消息渲染逻辑改写
- 线程存储、恢复机制或运行时审查后端逻辑重构

## 完成标准

- Agent 页面左侧只保留 `新对话 / 搜索 / 技能` 与 `最近对话`
- `设置` 不再出现在 Agent 工作台内
- `插件 / 自动化` 被移除
- `新对话` 点击后直接进入新线程
- `搜索` 与 `技能` 都以弹窗方式承接
- 右侧 inspector 只剩 `审查 / 文件 / 记忆`
- 页面中不再出现当前这些高重复、高装饰性的说明卡和快捷卡
- 页面整体第一眼感受应从“多功能入口页”变成“聊天工作台”
