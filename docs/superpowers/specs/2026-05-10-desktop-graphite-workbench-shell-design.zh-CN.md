# GoodNight 桌面 Graphite Workbench Shell 设计

## 目标

为 GoodNight 的无边框桌面端建立一套新的工作台壳层设计，方向固定为：

- 保留现有桌面骨架
- 使用 `Signature Workbench` 的整体气质
- 采用 `Graphite Solid` 的实体化窗口质感
- 去掉毛玻璃和模糊透视
- 同时优化标题栏拖拽、窗口移动、双击最大化/还原、右侧 AI pane 缩放的流畅度

这次工作不是重做整个产品，也不是重排业务页面，而是把桌面端最上层的窗口框体、标题栏、面板关系和交互手感统一到一个成熟的桌面产品标准。

## 已确认决策

- 总体方向：`Signature Workbench`
- 窗口质感：`Graphite Solid`
- 视觉禁用项：不使用毛玻璃，不使用 `backdrop-filter`，不依赖大面积透明叠层
- 气质目标：专业、耐看、稳定，有品牌识别但不刺眼
- 结构策略：保留左 rail / 顶部 titlebar / 中央主工作区 / 右侧 AI pane
- 实施重点：先打磨桌面壳层，再处理拖拽与缩放的流畅度

## 设计原则

### 1. 像桌面应用，不像网页壳子

窗口边界、标题栏、控件区、内容区和并列侧栏都要有明确层级，让整个界面更像一个完整的桌面产品，而不是若干网页模块拼起来。

### 2. 实体分层替代透明堆叠

质感由以下元素建立：

- 清晰的窗口框体
- 稳定的石墨灰表面层级
- 克制的描边
- 统一的阴影
- 少量功能性高光

不再依赖模糊、透视、玻璃泛白来制造高级感。

### 3. 品牌感只放在该出现的地方

品牌识别集中在：

- 标题栏整体语气
- 当前激活状态
- 少量角色色
- 局部高光和强调

主工作区保持中性和安静，避免大面积彩色背景分散注意力。

### 4. 手感优先于花哨动画

任何会影响窗口移动、拖拽缩放、点击响应的视觉效果都必须让位于流畅度。拖拽和缩放期间的首要目标是跟手、稳定、无顿挫。

## 布局与层级

继续沿用现有桌面骨架：

- 左侧主导航 rail
- 顶部无边框 titlebar / menubar
- 中央主工作区
- 右侧 AI companion pane

但要把这四部分收束成一个统一窗口，而不是多个彼此分离的区域。

### 左侧 rail

- 保持窄宽度和高密度
- 继续承担全局角色切换
- 视觉上与主窗口属于同一实体框体
- hover 和 active 使用统一的轻量强调，不出现突兀色块

### 顶部 titlebar

- 做成整套桌面端最有识别度的区域
- 高度更薄、更稳，接近原生桌面窗口头部
- 中央标题信息、菜单、项目切换和窗口按钮属于同一条清晰的操作带
- 不再看起来像单独漂浮的一条工具栏

### 中央主工作区

- 保持内容优先
- 作为最安静的视觉平面
- 让页面、知识、开发、测试等角色内容成为视觉主体

### 右侧 AI pane

- 保留第一公民地位
- 视觉上是并列工作面板，不是漂浮助手卡片
- 与中央主工作区平级，但通过宽度、描边和局部信息密度表达从属层级

## 视觉系统

### 整体风格

采用 `Graphite Solid`：

- 实体深灰框体
- 中性暗面表层
- 清晰的 1px 边界
- 统一而柔和的阴影
- 低刺激度对比

### 背景和面板

- 桌面窗口外层使用深石墨灰底
- 面板之间主要靠亮度差、边框和结构关系分层
- 避免在顶层使用强烈纹理、透明叠层和发白玻璃感

### 标题栏语言

- 标题栏保持克制，但必须比现在更完整
- 菜单区、标题区、工具区、窗口控制区使用同一套视觉语法
- 可以有非常轻微的局部品牌高光，但不做明显色带和渐变主导

### 控件语言

统一以下元素的尺寸、圆角、留白和状态反馈：

- 项目切换器
- 顶部按钮
- feature pill
- 窗口控制按钮
- 右侧 pane 的 resize handle

### 颜色策略

- 主色调以石墨灰、中性灰、低饱和亮面为主
- 品牌和角色色只作为点缀
- 不使用大面积蓝紫渐变和玻璃色叠加

## 交互与性能策略

### 标题栏拖拽

标题栏拖拽只保留一套原生路径。不要同时保留：

- `data-tauri-drag-region`
- 手动 `startDragging()`

在同一命中区域上的双重拖拽触发。

需要把标题栏命中区拆成两类：

- 可拖拽纯净区
- 可交互控件区

以下区域绝不参与拖拽判断：

- 菜单
- 按钮
- select / field
- 窗口控制按钮
- 其他带点击行为的交互控件

目标是降低误触、减少按下瞬间卡顿、让拖动和双击最大化更稳定。

### 双击最大化 / 还原

- 双击行为只挂在纯净拖拽区
- 双击触发路径和普通拖拽路径解耦
- 避免双击时与点击控件或拖拽起始逻辑互相干扰

### 右侧 AI pane 缩放

右侧 pane 缩放不应在每次 `pointermove` 时驱动整个 App 的 React 重渲染。

改为：

- 拖动过程中优先更新局部视觉宽度
- 松手后再提交最终 state
- 拖动期间关闭不必要的 transition
- 拖动结束再恢复常规过渡

这样可以让缩放更跟手，也避免主工作区跟着掉帧。

### Resize handle

- 命中区要更宽，便于抓取
- 可见线条要更细，避免存在感过强
- hover 和 active 反馈要更明确，但只在靠近或交互时出现

### 动效节奏

- hover / focus：`150ms - 180ms`
- pane 开合：`180ms - 220ms`
- 拖拽 / 缩放进行中：不做装饰性动画

任何动画都不能影响窗口移动、面板缩放或输入响应。

本轮不做：

- 各角色工作区内部组件的全面重构

## 验证标准

实现后必须满足以下结果：

1. 标题栏按下拖动没有明显顿挫，窗口移动更顺
2. 双击最大化 / 还原更稳定，不会和其他交互打架
3. 右侧 AI pane 拖拽缩放更跟手，拖动时明显更少掉帧
4. 整体桌面端比当前版本更完整、更统一、更像成熟桌面产品
5. 即使去掉毛玻璃，窗口仍然有明确质感和层级

## 实施顺序

1. 统一标题栏结构和命中区
2. 收紧桌面框体、边界、按钮和面板层级
3. 优化右侧 AI pane 的缩放路径与视觉反馈
4. 回归验证窗口拖拽、双击最大化和 pane resize 手感

## 完成标准

当以下条件全部满足时，这轮设计可视为落地完成：

- 无边框桌面端具备统一的 `Graphite Solid` 壳层气质
- 标题栏成为明确的产品识别区
- 毛玻璃相关体验完全退出顶层桌面壳层
- 窗口移动、拖拽和缩放手感明显优于当前版本
- 改动范围仍保持在桌面壳层，不牵连大面积业务页面重做
