# Agent Timeline Log Row 设计

## 摘要

本次设计将 AI 聊天区里的 `TimelineCard` 从“单行 chip 卡片”继续收敛为“线性日志行”。目标不是改运行时逻辑，而是在保持现有折叠/展开行为不变的前提下，让折叠态更接近参考图中的过程记录样式：一行一条、结构更线性、标签感更弱、主次更清晰。

## 目标

- 让时间线折叠态更像日志记录，而不是标签堆叠卡片。
- 把一条卡片压成真正的主阅读线：`阶段 / 标题 / 摘要 / 状态 / 详情`。
- 弱化 `phase`、`progress`、`status`、`toolCount` 的 chip 感，减少视觉噪音。
- 用左侧细竖线和文本分隔符代替卡片边框与胶囊标签。
- 保留现有详情抽屉行为，避免扩散到运行时事件建模和排序逻辑。

## 非目标

- 不修改 `timelineComposer` 的事件聚合逻辑。
- 不新增新的 timeline 交互。
- 不改详情抽屉的数据内容和展开层级。
- 不重做整个聊天消息列表的视觉体系，只微调 timeline 折叠态与其相关测试。

## 参考风格拆解

参考图的核心不是“更小的卡片”，而是“更像过程日志”：

- 左侧是细竖线和轻量前缀，用来标识阶段或动作类别。
- 中间是完整的一条语义线，用户第一眼先读到动作和摘要。
- 右侧是轻量状态与辅助操作，不抢主阅读权重。
- 容器存在感较弱，靠排布、分隔符和文字层级，而不是厚重面板感来组织信息。

## 当前问题

当前 `TimelineCard` 已经被压成单行，但仍保留较强的“胶囊标签卡片”特征：

- `phase`、`progress`、`status`、`toolCount` 都还是独立 chip，横向占位偏多。
- `查看详情` 仍像一个独立按钮，而不是行尾辅助操作。
- 容器圆角、padding、边框和阴影仍然偏像块级卡片。
- 标题与摘要虽然同列，但句子感还不够强，读起来更像一组并列元素。

## 设计决策

### 1. 折叠态改成线性日志行布局

折叠态继续保持单行优先，但从“多个并列标签”改成“单条日志句子”。内容顺序为：

- 左侧细竖线作为状态导轨
- `phase` 作为轻量前缀
- `title` 作为主语义锚点
- `summary` 作为紧随其后的说明文本
- `progress` 与 `toolCount` 仅在有值时显示，并通过轻分隔符并入主句
- `status` 与 `查看详情` 固定在右侧作为辅助信息

### 2. 去掉 chip 感

`phase`、`progress`、`status`、`toolCount` 不再强调饱满胶囊感：

- padding 清零或近似清零
- 字号略收紧
- border 和 background 去除，改成纯文本元信息
- `status` 优先表现为状态文字，而不是视觉块

### 3. 强化单句阅读节奏

`title` 与 `summary` 组成一条连续语义线：

- `title` 保持较强字重
- `summary` 更淡，并且只保留一行省略
- `progress`、`toolCount` 如果存在，应作为辅助上下文融入主线，不与标题争抢层级
- 使用 `/`、`·` 这类轻分隔符帮助用户把整行读成一句话

### 4. 右侧操作改为弱交互

`查看详情` 保留文字按钮，但视觉上改成尾部轻操作：

- 不再像主按钮
- hover 只做轻微强调
- 与状态并列，不占主内容心智

### 5. 保留详情抽屉

详情抽屉结构、展开方式、数据来源全部保留。此次只调整折叠态与抽屉入口之间的衔接密度，避免范围失控。

## 测试策略

- 用 source-level 测试锁定新的行内结构 class 名和旧结构的缺失。
- 用 CSS 回归测试锁定左侧细竖线、零圆角、纯文本元信息和单行布局特征。
- 继续跑现有 timeline 相关测试，确保抽屉和 bubble 卡片行为没有回归。

## 完成标准

- 折叠态时间线卡片在视觉上更接近参考图中的“线性日志行”。
- 卡片高度进一步降低，且卡片感明显减弱。
- `查看详情` 变成轻操作，不再像主按钮。
- 不影响详情抽屉展开。
- 相关 source-level 和 timeline 回归测试通过。
