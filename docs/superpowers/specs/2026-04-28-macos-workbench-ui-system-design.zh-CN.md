# GoodNight macOS 工作台 UI 统一设计

## 目标

为 GoodNight 建立一套可复用的 `macOS` 风格工作台 UI 标准，覆盖：

- 顶层工作台壳层
- 主侧栏、顶部工具栏、内容面板
- 按钮、输入框、选择器、标签、状态胶囊
- 弹窗与浮层的交互底座
- 浅色与深色双主题

本次目标不是一次重写所有业务页面，而是先建立统一基础层，并让所有顶层工作台先接入同一套视觉与交互标准。

## 已确认决策

- 风格方向：`Apple/macOS`
- 覆盖范围：全部顶层工作台
- 主题策略：浅色与深色同权
- 界面密度：紧凑型
- 主色策略：系统灰为主，每个工作台保留少量分色
- 技术策略：引入无样式底座能力，视觉样式全部自定义
- 底座选择：`Radix UI Primitives`

## 设计原则

### 1. 统一胜过局部惊艳

所有工作台先共享同一套字体、间距、圆角、阴影、边框和交互状态。业务区可以保留自己的布局，但不能再长出独立视觉系统。

### 2. 行为复用，视觉自有

`Radix` 只负责焦点管理、键盘交互、弹层定位和无障碍；按钮、弹窗、面板、工具栏等视觉外观完全由 GoodNight 自己定义。

### 3. 紧凑但不拥挤

默认字号统一为 `13px`，工具按钮、标签、输入框和工作台壳层采用紧凑密度，但保留明确的点击面积和可读性。

### 4. 浅深两套都成立

深色主题不是浅色反相。浅色侧重纸面感和轻边框；深色侧重层次和材质感。两套主题都需要有明确的表面层级、边框和状态反馈。

## 视觉标准

### 字体

- 基础字体：`-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'SF Pro Display', 'Helvetica Neue', Arial, sans-serif`
- 默认字号：`13px`
- 工具栏标签与辅助文案：`11px - 12px`
- 主标题保留较大字号，但进入统一的字阶体系

### 间距与圆角

- 紧凑间距基线：`4 / 6 / 8 / 10 / 12 / 16 / 20`
- 小控件圆角：`8px`
- 常规面板圆角：`12px`
- 浮层与高级面板圆角：`14px - 16px`
- 胶囊控件：`999px`

### 表面层级

- `window`：应用窗口背景
- `chrome`：侧栏、标题栏、工具栏
- `panel`：主要内容面板
- `panel-elevated`：浮起的卡片、二级面板
- `overlay`：弹窗、下拉、提示层

### 工作台分色

- `knowledge`：蓝色偏冷静
- `wiki`：青色偏结构
- `page`：靛蓝偏编辑
- `design`：紫粉偏创作
- `develop`：绿色偏构建
- `test`：橙色偏验证
- `operations`：红橙偏发布

这些颜色只用于：

- 当前激活态
- 角色标识
- 少量状态强调

不用于大面积背景涂色。

## 组件标准

### Button

- `primary`：主要动作，实心，macOS 风格轻光泽
- `secondary`：次要动作，半透明表面
- `ghost`：工具栏和列表内轻动作
- `danger`：危险动作，严格限于删除、重置等场景
- `icon`：方形紧凑图标按钮

### Input / Select / Textarea

- 统一高度、统一边框、统一 focus ring
- 使用更克制的背景，不再让不同页面各自定义输入框材质
- 原生 `select` 保留，但包裹在统一壳层中

### Panel

- 所有顶层工作台内容容器统一到同一套 `panel` 语义
- 标题区、工具区、正文区的分隔关系统一

### Dialog / Menu / Tooltip

- 由 `Radix` 提供行为底座
- 统一使用 GoodNight 自己的窗口样式、阴影和边框
- 弹窗遵守与工作台一致的标题栏和操作区规范

## 架构设计

### Foundation

新增一层 UI foundation，集中管理：

- 主题 token
- 浅深主题变量
- 工作台角色强调色
- 共享控件 class

### Primitives

新增一组基础组件：

- `MacButton`
- `MacIconButton`
- `MacPanel`
- `MacField`
- `MacSelect`
- `MacDialog`

### 顶层接入

本轮优先改造：

- 桌面主壳层
- 顶部菜单与工具栏
- 左侧主导航
- 项目管理入口页
- 各顶层工作台共享容器

业务区深层细节暂不一次改完，但必须能吃到新的字体、颜色、边框和状态语义。

## 测试策略

- 增加一条 UI 系统回归测试，验证：
  - 新 foundation 文件存在
  - `Radix` 依赖已经接入
  - `App.tsx` 已使用新组件
  - `ProjectSetup.tsx` 已使用新组件
  - `App.css` 已声明 macOS 主题 token
- 运行现有桌面工作台相关测试，确保顶层壳层未回退
- 运行构建，确认样式和组件改造未破坏打包

## 范围外

本轮不做：

- 全量替换所有业务区深层按钮和局部组件
- 把所有自定义控件都改成 Radix
- 重新设计信息架构或工作台布局逻辑

## 完成标准

- 顶层工作台具备统一 macOS 壳层观感
- 默认字体统一为 `13px`
- 浅深主题都具备一致的控件表现
- 项目入口页、顶部工具栏、主侧栏、主要按钮和表单已明显统一
- 新的基础组件和 token 可供后续页面继续复用
