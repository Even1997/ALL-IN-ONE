Phase 5 Delivery Map

This section maps the actual Phase 5 implementation to files and architectural meaning.

Gateway foundation

src/modules/ai/runtime/conversation/runtimeConversationGateway.ts and useRuntimeConversationGateway.ts now own session reconciliation, active-session selection, thread routing, and runtime projection assembly.

  • One shared selection model for active session and thread ids.
  • One projection for messages, tools, approvals, recovery, live state, and background tasks.
  • Explicitly an in-repo boundary, not a fake sidecar claim.

Embedded chat adoption

src/components/workspace/AIChat.tsx now reads active conversation state from the gateway and uses shared runtime-thread reconciliation.

  • Session selection and bootstrap logic no longer live as a second custom derivation.
  • Replay recovery, latest turn, live state, and background task reads stay aligned with shell views.

GN shell adoption

src/components/ai/gn-agent-shell/GNAgentChatPage.tsx now centralizes shell runtime reads, while GNAgentThreadList.tsx and GNAgentStatusPanel.tsx are prop-driven.

  • The shell no longer rebuilds shared state through separate store subscriptions.
  • Thread selection, resume, status, and recent activity stay tied to the same projection.

Verification and closure

Phase 5 now has dedicated source and gateway tests, plus the broader shell-facing regression checks required for this phase.

Conclusion

The product is no longer in the "patchy chat panel" stage. It now has a coherent in-app runtime model.

What is truly closed now

  • One event plane for assistant UI stability.
  • One session event log for restore and replay.
  • One execution hierarchy for local, team, and workflow runs.
  • One lifecycle path for skills, MCP, approvals, and memory.
  • One conversation gateway shared by AIChat and GN shell.

What is intentionally not claimed yet

  • No true external sidecar process yet.
  • No detached CLI or TUI runtime entry point yet.
  • No remote adapter layer yet.
  • No computer-use or broader platform plugin surface yet.
