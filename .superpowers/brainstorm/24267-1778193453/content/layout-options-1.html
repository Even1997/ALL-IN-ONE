Agent 工作台改版方向

目标是把聊天舞台做成主角，减少左中右同时抢注意力的问题。

A. 聊天优先 + 右侧停靠审查
Rail
Threads
精简标题栏
主聊天区
Composer
Inspector

推荐：聊天优先 + 右侧停靠

保留三栏能力，但把左侧缩成导航和线程，中间变成绝对主舞台，右侧默认保留审查。

优点

  • 改动相对克制，风险低
  • 仍然保留 Inspector 的工具价值
  • 最容易把当前页面从“乱”拉回“专注”

代价

  • 右侧仍会占一部分宽度
B. 聊天优先 + 审查抽屉
Rail
Threads
轻量标题栏
全宽主聊天区
Composer
浮层 Inspector

更激进：聊天全宽 + Inspector 抽屉

默认几乎只看聊天，需要审查时再从右侧拉出，沉浸感最强。

优点

  • 视觉最干净
  • 最接近 Codex / Claude 工作流

代价

  • 需要重新处理浮层与聊天的关系
  • 交互改动更大
C. 单侧边栏 + 聊天主舞台
合并导航 / 线程
标题栏 + 状态
聊天区
Composer

最简：只保留一侧边栏

把 rail、threads、inspector 进一步合并，页面会更像一个单主舞台应用。

优点

  • 结构最简单
  • 页面会明显更清爽

代价

  • 会触碰现有信息架构
  • 开发范围最大
