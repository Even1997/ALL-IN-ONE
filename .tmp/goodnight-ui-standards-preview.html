GoodNight UI Standard

沿用当前风格，补齐规范

这不是新视觉方向，而是把现在的 macOS 工作台质感变成可执行规则。

推荐基线

保留当前外观：轻玻璃、紧凑密度、角色色点缀、14px 面板圆角。主要修“不统一”：字号、间距、卡片嵌套、按钮状态。

01 / Tokens

基础尺度

字体

11px标签、eyebrow、弱元信息，必须大写时加 0.04em 字距。
12px说明文字、列表副标题、状态描述。
13px默认 UI 字号。按钮、表单、正文控件统一基准。
16-18px面板标题和工作台标题。不要在工具界面里用 hero 字号。

间距

4px内部微间距、图标和文字之间。
8px按钮组、紧凑列表、同类控件之间。
10-12px工作台 gutter、卡片间距、模块之间。
14-16px面板内边距、表单区块、弹窗主体。

圆角

6px小型 toggle、菜单项、代码/状态小块。
10px按钮、输入框、顶部工具按钮。
12px卡片、列表节点、局部容器。
14px主面板、顶部栏、侧边栏。全局工作台默认值。

阴影和层级

panel0 18px 36px，给主面板和顶部栏。
floating0 24px 56px，给菜单、弹窗、悬浮 AI。
focus3px 半透明角色色，不使用 outline 默认蓝框。

04 / Rules

执行规则

应该统一

  • 所有工具表面优先使用 13px 基准字号。
  • 主面板和工具栏统一 14px 圆角。
  • 按钮最小高度 32px，输入框最小高度 36px。
  • 同级面板间距使用 10-12px，不随手写 17/19/23px。
  • hover 只做轻微上移、边框、背景变化。

应该避免

  • 不要再新增 18px 以上的大圆角工具卡片，除非是首屏/弹窗。
  • 不要卡片套卡片套卡片，重复容器要降成分隔线或列表。
  • 不要每个模块发明自己的按钮高度、阴影、标签样式。
  • 不要用大面积渐变来表达层级，层级交给面板、边框、留白。
