
Visual Vibe Coding Studio
系统设计文档（更新版）
1 项目背景

当前 Vibe Coding 主要通过自然语言驱动 AI 编写代码，但存在以下问题：

表达歧义
Token 消耗高
修改成本高
用户无法直观看到系统结构

因此本系统目标是：

构建一个可视化 Vibe Coding 平台

通过：

可视化需求
AI Agent
结构化项目图谱

实现 从需求到部署的自动化软件生产流程。

2 核心设计理念

系统围绕三个核心理念：

1 Visual Spec（可视化需求）
Visual Spec → Code

而不是

Text → Code

Visual Spec 包含：

Feature Tree
Page Structure
Wireframe
UI Design
2 Human in the Loop

系统始终保持 AI + 用户协作。

AI生成
↓
用户调整
↓
AI继续生成

用户可以直接：

拖拽组件
修改结构
删除模块
3 Project Graph（项目图谱）

系统所有信息都会结构化。

Project
 ├ Requirement
 ├ Feature
 ├ Page
 ├ Component
 ├ API
 ├ Database
 └ Test

AI只读取相关节点。

4 项目管理模块（Project Manager）

项目管理模块是 整个系统的入口。

用户创建项目时，需要先确定：

产品类型
技术栈
框架
部署方式

这些信息会成为 AI生成代码的基础上下文。

↓
选择框架
