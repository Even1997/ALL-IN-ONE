Standard Page

GoodNight Desktop Component Standard

单页组件目录。桌面原生、安静、可替换。

◫ Sourcesingle-page
⌘ Tonedesktop native
◐ Themelight / dark
▣ Replacemodule first
1. 先模板

每个页面先归属到 document、canvas、agent、quality、operations 等模板。

2. 再 primitive

中心区域只能选择一个主 primitive：note surface、directory tree、canvas 或 chat stage。

3. 组件组合

业务模块传内容和状态，不直接发明新卡片、新按钮、新面板。

4. 状态完整

默认、hover、selected、collapsed、loading、empty、error、confirm 必须有一致表现。

00.1 Reference Model

Ant Design 给我们的组织方式

Ant Design 的关键不是某个按钮长什么样，而是把组件系统做成“目录、示例、API、Token、状态”的可执行文档。

分类入口

组件按通用、布局、导航、数据录入、数据展示、反馈等职能分组，开发者先按任务找组件。

catalogsearchable
组件详情

每个组件都有独立说明页，固定展示使用场景、代码演示、API、语义结构和主题 Token。

usageapi
状态闭环

组件不只展示默认样子，还要覆盖 disabled、loading、selected、error、empty、compact 等状态。

statesvariants
Token 驱动

视觉变化通过主题变量和组件 Token 收敛，业务模块调用组件，不再散落硬编码样式。

单页内每个组件模块必须包含的标准块

这就是我们自己的 Ant Design 式组件详情格式，但全部收在一个页面里。模块落地时，缺任意一块都不算真正接入标准。

BlockRequired Content
When To Use适用模块、禁止场景、是否是主工作面。
Anatomy区域结构、插槽、密度、可折叠或可调整边界。
Statesdefault / hover / active / selected / loading / empty / error / disabled。
APIprops、事件、slot、受控状态、默认值。
Tokens颜色、间距、圆角、边框、焦点、暗色主题映射。

  tokens={["--wb-surface-strong", "--wb-line", "--wb-focus-ring"]}

代码层统一使用 workbench components 和 tokens。模块不得直接复制标准页 HTML，只引用组件、状态和交互契约。

LayerShould UseShould Avoid
Themesrc/styles/workbench/tokens.css组件内硬编码 hex / rgba
NavigationDirectoryTree, WorkbenchList, shared sidebar rows把导航区做成卡片墙或自定义一套 hover/selected
SurfaceNoteSurface, WorkbenchPanel, StateCardrounded card stacks / dashboard chrome
AIAIComposer, AILaneStack, AIToolExecutionCard, AIConfirmCard把工具、思考、最终答案合成一个 bubble

const template = getDesktopWorkbenchModuleTemplate(currentRole);
