Findings

Where The UI Drifts From The Standard

These are presentation-layer gaps. None require changing provider adapters, canonical event semantics, tool truth, or replay meaning.

render model lane taxonomy
Tool execution is not a first-class lane

Tool trace cards exist visually, but the output model classifies them as generic bubbles. The standard asks for an explicit tool_execution card/lane sourced from runtime events.

confirm approval
Approval and question cards are inserted as bubbles

StateCard improves the surface, but confirm_or_next_step is still not represented in message ordering. This makes risky decisions feel like another timeline card instead of a distinct decision lane.

visual grammar chat residue
User input still reads like chat UI

The right-aligned user bubble conflicts with the standard's compact source-record pattern. It should anchor the turn like a note row, not compete with the document surface.

states coverage
State coverage is partial

Approval/question cards use StateCard and wb-state polish, but thinking, final, tool, loading, collapsed, and error variants do not share one state-card vocabulary.

tokens CSS
Old token families still leak through

The CSS mixes mode vars, gn-agent vars, and wb vars. Later overrides calm the page, but the source still carries conflicting radius, hover, and surface rules.

architecture safe layer
The fix belongs above runtime truth

Runtime facts already exist. The correct layer is projection and UI composition: expand render lanes, remap ordering, then restyle surfaces to match the workbench standard.

Fix Direction

Recommended Implementation Path

The safest path is incremental: preserve runtime facts, introduce a richer display taxonomy, then update the card families.

01 Expand lane kinds

Add user_input, thinking, feedback, tool_execution, final_answer, and confirm_or_next_step to the message render model.

02 Map runtime cards by meaning

Convert tool traces to tool_execution and approval/question to confirm_or_next_step without reparsing assistant text.

03 Unify card surfaces

Create AI lane card styles that mirror ui-standards: compact user, margin-note thinking, tool facts, document final, confirm card.

04 Verify states and themes

Cover collapsed, loading, error, hover, selected, and dark mode for every lane family before changing runtime behavior.
