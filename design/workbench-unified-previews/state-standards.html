State Contract

One Shared Language

Required States
default hover selected collapsed empty loading error
  • These states must be designed intentionally, not patched in later
  • Every major workbench surface should use the same visual logic for status and emphasis
  • Selection is stronger than hover, but should still feel quiet and desktop-native
Behavior Rules
defaultCalm resting state with clear hierarchy and no unnecessary emphasis.
hoverSubtle tint, shadow, or lift only. Never use loud glow or bubble chrome.
selectedPersistent focus state for the current file, node, note, or AI result.
collapsedKeep identity, summary, and status. Remove depth, not meaning.
emptyQuiet starter state with one next action and clear context.
loadingUse pulse, skeleton, or progress. Avoid decorative shimmer walls.
errorExplain the issue inline and provide the recovery path inside the same surface.
Motion Rules
Default rest
Hover 120-160ms
Selected persistent
Error and empty stay static
  • Hover and select motion should feel Finder / Notes-like, not playful
  • Loading uses pulse or progress only, while error and empty rely on clarity rather than motion
  • Dark mode must preserve the same interaction hierarchy, not invent a different one

AI Contract

Machine-Readable State Rules

state_standard:
  required_states:
    - default
    - hover
    - selected
    - collapsed
    - empty
    - loading
    - error
  surfaces:
    directory_tree:
      state_meaning:
        default: "flat rows, neutral hierarchy"
        hover: "subtle tint or row lift"
        selected: "persistent current file/folder state"
        collapsed: "disclosure closed, identity remains"
        empty: "no results or no children with next action"
        loading: "child rows use skeleton or pulse"
        error: "row-local retry and reason"
    note_surface:
      state_meaning:
        default: "clean reading and writing page"
        hover: "light action affordance only"
        selected: "active block or current note"
        collapsed: "summary row with key metadata"
        empty: "starter prompt with one primary next action"
        loading: "preserved layout with text skeleton"
        error: "inline conflict or sync status with recovery"
    infinite_canvas:
      state_meaning:
        default: "open stage"
        hover: "node hover or edge emphasis"
        selected: "chosen node or cluster"
        collapsed: "cluster summary remains visible"
        empty: "starter state inside stage"
        loading: "graph or node hydration in place"
        error: "overlay or inspector-local recovery"
    ai_companion:
      state_meaning:
        default: "quiet support panel"
        hover: "action discovery only"
        selected: "active result or inspector focus"
        collapsed: "one-line summary of long thinking or tool output"
        empty: "ready prompt before work starts"
        loading: "progress or pulse, no prose wall"
        error: "specific failure plus retry path"
  interaction_rules:
    hover: "120-160ms ease-out"
    selected: "persistent, stronger than hover"
    loading: "pulse, skeleton, or progress only"
    error: "avoid animation unless needed for immediate clarity"
  design_rules:
    - "keep a native desktop tone"
    - "avoid chat bubbles and loud dashboard framing"
    - "use the same hierarchy in light and dark mode"
    - "make empty and error states first-class design work"
